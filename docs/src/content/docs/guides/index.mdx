Use Legend-State as the first state model in app code.

`use-legend` is an observable-first React utility layer built on Legend-State:
state lives in observables, React renders the tree, and fine-grained boundaries
update only the UI reads that changed.

## Observable-First Rules

- No `useState`
- No `useReducer`
- No component-wide re-rendering as the default update path

Instead, keep mutable values in observables and update only the UI nodes that depend on those values.
Hooks are still useful, but mainly for composing side effects, timers, sensors, and reusable logic around observables.

## Why hooks + observables work well together

When an interval updates a regular React state value, the whole component re-renders.
When the interval updates an observable, only the bound text node updates and the parent component render count stays stable.
